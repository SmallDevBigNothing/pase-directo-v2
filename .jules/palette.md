## 2025-06-30 - Fix Custom Checkbox Accessibility
**Learning:** Custom toggle switches in the admin panel were built using `display: none;` on the native `<input type="checkbox">`, which completely removed them from the accessibility tree and broke keyboard navigation (`Tab` key). Also, the input lacked `aria-label` which meant screen-readers had no context for the control. When visually hiding inputs for custom styling, they should use `opacity: 0; width: 0; height: 0; position: absolute; outline: none;` instead, so they remain focusable.
**Action:** When building or fixing custom form controls, always ensure the native input remains visually hidden but focusable, and use `:focus-visible` on adjacent visual elements (like `.toggle-slider`) to display a clear focus ring. Ensure an `aria-label` is always provided if there is no explicit `<label>` text.
