## 2025-02-18 - XSS in inline event handlers via template literals
**Vulnerability:** Inline event handlers (like `onclick="..."`) in the dynamically generated HTML files used template literals with string replacements which could be broken out of if user-controlled input contained a quote character. This led to arbitrary JS injection within the DOM context.
**Learning:** The Express application dynamically generates HTML without a templating engine by relying on JavaScript template literals in a few modules. Because data is formatted straight into these strings, injecting complex objects directly into an HTML attribute is extremely dangerous if not stringified and encoded correctly.
**Prevention:** When injecting objects or parameters into `onclick` handlers, ALWAYS serialize the data using `JSON.stringify` and encode it using `escapeHtml(...)`. Do NOT rely on `.replace(/'/g, "\\'")` which is brittle. Avoid string templates within HTML event attributes when possible.
