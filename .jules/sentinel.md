## 2024-06-28 - Removed Hardcoded Secrets & Admin Authentication Bypass Prevention
**Vulnerability:** The application was falling back to hardcoded secrets for `SESSION_SECRET` and `ADMIN_PASSWORD` if environment variables were not set. Removing the fallback for `ADMIN_PASSWORD` without explicit truthiness checks could lead to an authentication bypass where an empty password payload matches an undefined `process.env.ADMIN_PASSWORD`.
**Learning:** Hardcoded fallbacks are risky, especially for admin access or production secrets. Removing them safely requires carefully evaluating the conditions (e.g. truthiness of undefined matching undefined) to prevent catastrophic bypasses.
**Prevention:** Always use truthiness checks for environment variable secrets (e.g., `if (adminPassword && password === adminPassword)`). Throw descriptive errors during startup in production for missing required secrets (like `SESSION_SECRET`). Ensure tests run to verify that empty payloads don't accidentally match undefined environments.
